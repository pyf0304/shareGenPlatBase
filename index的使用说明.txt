该文件用于说明项目根目录下 index.html 的使用方式。

一、文件作用
index.html 是项目的页面入口文件，主要负责：
1. 定义基础的 HTML 结构。
2. 挂载 Vue 应用的根节点 `<div id="app"></div>`。
3. 根据不同使用场景，引入不同的入口脚本。

二、当前文件中的两种入口方式
index.html 中已经保留了两套脚本引用方式：

1. 现场开发环境
使用：
`<script type="module" src="/src/main.ts"></script>`

说明：
- 该方式用于当前项目单独运行、调试和开发。
- `src/main.ts` 会创建 Vue 应用实例，并挂载路由、Pinia、Element Plus 等运行时能力。
- 适合执行 `pnpm dev` 后在浏览器中直接访问页面。

2. 生产接入环境
使用：
`<!-- <script type="module" src="/src/index.ts"></script> -->`

说明：
- 该方式用于把当前项目作为能力模块或组件库提供给其他系统接入。
- `src/index.ts` 主要负责统一导出功能、页面模块、方法和类型，不负责像 `main.ts` 那样直接启动整套页面应用。
- 当项目以“被集成”的形式使用时，应按接入要求启用该入口或基于该文件进行构建输出。

三、如何切换
如果是本地开发、联调、查看页面效果：
- 保留 `main.ts` 的 script。
- 不要启用 `index.ts` 的 script。

如果是为外部项目提供发布能力或做生产集成：
- 根据实际接入方式处理 `index.ts`。
- 一般不直接同时启用 `main.ts` 和 `index.ts`，避免入口职责冲突。

四、推荐做法
开发时建议使用：
- `pnpm dev`

构建时建议使用：
- `pnpm build`

构建完成后：
- 由 Vite 产出 dist 目录内容。
- 如需发布给其他项目使用，应结合 `package.json` 中的 `main`、`types` 和构建产物进行集成。

五、注意事项
1. `main.ts` 是“启动应用”的入口，适合独立运行。
2. `index.ts` 是“对外导出”的入口，更适合封装后供外部引用。
3. 修改 index.html 时，重点确认当前场景到底是“页面开发”还是“模块集成”。
4. 若后续需要接入 CDN、外部样式或现场库文件，也通常在 index.html 中补充对应的 `<link>` 或 `<script>` 标签。

六、示例
本地开发示例：
```html
<script type="module" src="/src/main.ts"></script>
```

生产集成场景可参考：
```html
<!-- <script type="module" src="/src/index.ts"></script> -->
```

七、结论
index.html 的核心用途不是承载业务逻辑，而是决定项目启动入口。
通常情况下：
- 调试运行用 `src/main.ts`
- 对外集成用 `src/index.ts`

后续如果项目入口策略调整，建议同步更新本说明文件，避免开发和发布时混用入口文件。
